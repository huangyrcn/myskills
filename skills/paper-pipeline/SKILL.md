# Paper Pipeline

Run the end-to-end paper workflow.

## Pipeline Order

Default pipeline (resolve → acquire → repo):

```
paper-resolve     → Determine canonical identity → metadata.yaml
        ↓
paper-acquire     → Download PDF, normalize to paper.md → assets/
        ↓
paper-repo        → Discover repo, clone if found → repo_search + repo/
```

**Reading notes are NOT generated by default**. Use `--with-notes` to enable:

```
paper-reading-notes → Generate research note → reading-note.md
```

Do not skip stages. Each stage depends on the previous one's output.

## Stage Contracts

### Stage 1: paper-resolve

**Input**: Paper reference (title, DOI, arXiv ID, URL, method name)

**Output**: `~/papers/{title_slug}/metadata.yaml`

```yaml
identity:
  canonical_url: "..."
  primary_id: {type, value}
  aliases: {doi, arxiv, openalex, semantic_scholar, ...}
bibliography:
  authors: [...]
  year: ...
  venue: "..."
urls:
  canonical: "..."
  pdf: "..."
acquisition_hints:
  pdf_sources: [...]
```

**Skip if**: `metadata.yaml` already exists with `identity` section.

### Stage 2: paper-acquire

**Input**: `metadata.yaml` from resolve

**Output**: 
- `paper/paper.pdf`
- `paper/paper.md`
- `paper/paper_images/`

Uses `acquisition_hints.pdf_sources` priority order:
1. arXiv PDF (free)
2. PMC PDF (free)
3. Semantic Scholar OA PDF
4. Publisher PDF (may need auth)

**Skip if**: `paper/paper.md` already exists and user doesn't request refresh.

### Stage 3: paper-repo

**Input**: `metadata.yaml`, `paper/paper.md`, `paper/paper.pdf`

**Output**: 
- `metadata.yaml` → `repo_search`
- `repo/` → cloned repository (if confidence >= medium)

```yaml
repo_search:
  selected:
    url: "https://github.com/..."
    confidence: "high"
    type: "official"
    cloned_to: "repo/"
  candidates: [...]
```

**Skip if**: 
- `repo_search.selected` already populated
- User passes `--skip-repo` flag

### Stage 4: paper-reading-notes (OPT-IN)

**Input**: Complete raw bundle (metadata + paper + repo info)

**Output**: `reading-note.md`

**Trigger**: Only runs when `--with-notes` flag is provided.

**Skip if**: User passes output path that already exists and doesn't request overwrite.

## Storage Contract

```
~/papers/{title_slug}/
  metadata.yaml          # identity, bibliography, urls, acquisition_hints, assets, repo_search
  paper/
    paper.pdf
    paper.md
    paper_images/
    latex/
      main.tex
      ...
  repo/                  # cloned repo (optional, manual)
```

Note output:
1. User-specified `--output` path
2. `./papers/{title_slug}/reading-note.md` when cwd is not `~`
3. `~/tmp/paper-notes/{title_slug}/reading-note.md` when cwd is `~`

## Re-entry Policy

Treat the workflow as incremental:

| Condition | Action |
|-----------|--------|
| `metadata.yaml` exists | Skip resolve, proceed to acquire |
| `paper/paper.pdf` exists | Skip download, check if MD needed |
| `paper/paper.md` exists | Skip normalization unless refresh requested |
| `repo_search.selected` exists | Skip repo discovery unless refresh requested |
| Note exists | Skip note generation unless overwrite requested |

## Network Policy

Use `web-kit` skill for all network operations:

```bash
# Resolve stage
ask-search "query" -e google -n 10
ask-search "query" -e openalex -n 5
ask-search "query" -e "semantic scholar" -n 5

# Acquire stage
wget "https://arxiv.org/pdf/xxx.pdf"
${SKILL_DIR}/../web-kit/scripts/crwlr crawl -o md "{url}"

# Repo stage
ask-search "{title} github" -e google -n 10
```

## Example Invocation

```bash
# Full pipeline (resolve + acquire + repo) - NO notes by default
python3 "${SKILL_DIR}/scripts/run_pipeline.py" "SpatialGlue spatial multi-omics"

# From DOI
python3 "${SKILL_DIR}/scripts/run_pipeline.py" "10.1038/s41592-024-02316-4"

# From arXiv ID
python3 "${SKILL_DIR}/scripts/run_pipeline.py" "arxiv:1704.01212"

# Skip repo discovery
python3 "${SKILL_DIR}/scripts/run_pipeline.py" "SpatialGlue" --skip-repo

# Include reading notes (opt-in)
python3 "${SKILL_DIR}/scripts/run_pipeline.py" "SpatialGlue" --with-notes

# Specify note output path
python3 "${SKILL_DIR}/scripts/run_pipeline.py" "SpatialGlue" --with-notes --output ./notes/spatialglue.md

# Refresh existing
python3 "${SKILL_DIR}/scripts/run_pipeline.py" "SpatialGlue" --refresh
```

## Error Handling

| Error | Recovery |
|-------|----------|
| Resolve fails | Ask user for clarification, suggest candidates |
| PDF download fails | Try next source in priority order |
| LaTeX unavailable | Fall back to PDF → MinerU |
| Repo not found | Continue to notes, skip Code Alignment section |
| Note generation fails | Save intermediate state, allow resume |

## Dependencies

- `web-kit` — Network operations
- `paper-resolve` — Identity resolution
- `paper-acquire` — PDF/LaTeX acquisition
- `paper-repo` — Repository discovery
- `paper-reading-notes` — Note generation
- `pdf-to-md` — PDF normalization
